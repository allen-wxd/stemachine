Overview

STEMachine is a learning platform for students to practice Python coding and project development. STEMachine delivers an engaging learning experience with the help of a mechanical arm that students can control via Python coding. Through creating and designing projects that manipulate the mechanical arm in different ways, students can discover the fun in coding and project development.

This document will help you learn how to use STEMachine to create projects, start coding and eventually make the mechanical arm do your bidding all by yourself. For each of your projects, STEMachine will also generate a video that records the movements of the mechanical arm under the guidance of your code. You can watch the video online or download it and show it off to your friends and family.

You can follow the steps below to start your first adventure into project development and go through the entire development process. Click a step to view details.

Login

Open your browser and go to the STEMachine login page:

http://STEMachine address/htm/login.html

Please replace "STEMachine address" with your school's actual STEMachine server address. If you do not know the address, please consult your teacher.

Note: It is recommended that you use Chrome browser to access STEMachine.

Enter your account and password. Your account is your student ID. The default password is 123456. It is recommended that you change your password upon your first login.

You can check Remember password to let the browser remember your password. Click Sign in.

Change Password

  1. On the login page, click Change password.
  2. In the Username field, enter your account (i.e., your student ID). Enter your old password (the default password is 123456) and then enter your new password twice.
  3. Click Submit.

Forgot Password

If you have forgotten your password, please contact your teacher or system administrator. They can reset the password for you.

Project Management

Before you upload your code, please create a project. You can create multiple projects that serve different design purposes. You can generate a result demonstration video for each project after it is finished.

Click Code or Results to switch between code view and results view. In the left pane, you can see a list of your projects and their status.

Status Description
Unpublished
 A piece of Python code has been uploaded, but has not been published yet.
Processing
 A piece of Python code has been uploaded and published. The system is still processing the code and results have not come out yet.
Unwatched
 The results video has been generated, but has not been played yet.
Watched
 The results video has been generated and played.
Error
 A piece of Python code has been uploaded, but the code cannot be executed due to errors.

Upload Code

  1. Go to Code.
  2. In the left pane, select the project you want to upload code to.
  3. In the right pane, click Upload code.
  4. Click Choose File and select the .txt or .py file that contains the Python code you wrote.
  5. Please wait for the file to upload. Afterwards, the file name you uploaded will appear on the right side of the Choose File button. Click Upload.

You may want to:

  • Create a Project
  • Go back to Code Management
  • Go back to the top

Syntax Check

  1. After you have uploaded a piece of code, click Syntax check.
  2. If errors are found, a window will pop up displaying the line numbers of the errors and their details. If no error is found, there will be no pop-up window, and the "Publish code" button will turn yellow, indicating the code is ready for publication.
  3. If errors were found during the syntax check, you can edit the code directly on the webpage. After you have finished editing the code, click Syntax check again to perform a second check. You can repeat this process until there is no error in your code. Afterwards, click Publish code to save changes and publish the edited code. If you leave the page without clicking Publish code, a dialogue box will pop up asking if you want to save changes. Click OK to save changes. If you click Cancel, the changes you made will not be saved.

You may want to:

  • Upload Code
  • Go back to Code Management
  • Go back to the top

Publish Code

  1. After you have passed the syntax check, the Publish code button will turn from grey to yellow. Click it and then click "OK" to publish your code.

Note: You can only publish your code after it has passed the syntax check.

Results Demonstration

After a piece of code is published, the system will try to control the mechanical arm by executing the code. The movements of the mechanical arm will then be recorded in a video.

At the top of the page, click Results to view results.

Play Video

If the code of a project has been successfully executed, there will be a "Watched" or "Unwatched" label at the bottom-right corner of the project indicating a video has been generated. Select a project and click the Results button at the top of the page to play the result video.

Download Video

Click at the bottom-right corner of the video pane to download it.

If you have published a piece of code but cannot play the project video, it may be due to one of the following reasons:

  • The code is still under process and the results have not been generated yet. You can check back later.
  • An error has occurred during the execution of the code.

In case of an execution error, you can click Code at the top of the page and check the project status in the left pane.

Status Description
Unpublished
 A piece of Python code has been uploaded, but has not been published yet.
Processing
 A piece of Python code has been uploaded and published. The system is still processing the code and results have not come out yet.
Unwatched
 The results video has been generated, but has not been played yet.
Watched
 The results video has been generated and played.
Error
 A piece of Python code has been uploaded, but the code cannot be executed due to errors.

If the project status is "Error", it means the system encountered an error while executing the project code. You can click the project to edit the code. After you have fixed the code, run a syntax check and publish it again.
